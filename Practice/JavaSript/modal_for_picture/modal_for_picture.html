Разбор

Начнем с разбота кода HTML

Здесь у нас есть два блок. В первом блоке (class="box") мы поместили три тега img с классом my__img.

Второй блок собственно и будет нашим можальным окном. Этому блоку мы дали id=my__modal и class=my__modal. В этом блоке у нас содержатся кнопка закрытия (span class="close" id="close) и блок в который будет помещаться сама фотография (class="modal__block" id="modal__block") и блок с надписью (id=caption)

Для чего сделали блок modal__block в блоке my__modal? для того что бы кнопка закрытия не залезала на фотографию

Теперь рассмотрим CSS файл. Файл большой, поэтому разделим его на три части.

Рассмотрим первую чать. В первом блоке мы указываем нулевый внешние и внутренние отступы у всех элементов страницы

.box - стилизуем блок в котором хранятся наши фотографии

  • display: flex; - делаем блок флексом
  • flex-direction: row; - задаем выравнивание флексов в линию вдоль основной оси
  • flex-wrap: wrap; - флексы могут выравниваться в несколько строк
  • justify-content: flex-start; - флексы выравниваются слева
  • align-items: center; - флексы выравниваются по центры внутри контейнера(бесполезное свойство в этом случае так как вы выравнили флексы слева в предыдущем свойстве)
  • align-content: center; - выравниваем контент по центру (опять же не понятно зачем так как мы выровнили их по левой стороне) Если мы хотим флексы выравнить по центру то в свойстве justify-content: надо было указать center
  • width: 100%; - ширина контейнера 100% от ширины экрана.



.my__img - сами картинки

  • width: 300px; - фиксируем ширину картинки
  • height: auto; - в соответствии с шириной высота высчитывается автоматически
  • object-fit: cover; - это очень полезное свойство применяется в случаях отображения картинки или видео в условиях когда заданы размеры области с изображением которые не совпадают с размерами самого изображения или видео, это свойство позволяет заполнить область картинкой с сохранением пропорция картинки, тоесть она не исказится
  • cursor: pointer; - указали курсор - pointer (ладошка с пальцем)

.my__img:hover - при наведении на картинку она примет значение прозрачности 0.7.

.my__modal - стилизуем модальное окно

  • display: none; - отключаем блок
  • position: fixed; - фиксируем позицию
  • z-index: 1; - устанавливаем его поверх все окон
  • right: 0; - расстояние до правой границы
  • left: 0; - расстояние до левой границы
  • top: 0; - расстояние до верхней границы
  • width: 100vw; - ширина 100%
  • height: 100vh; - высота 100%
  • overflow: hidden; - скрываем текст если он выходит за пределы блока
  • background-color: rgb(0,0,0); - цвет фона в режиме rgb
  • background-color: rgba(0,0,0,.7); - не совсем понял зачем надо было этот же цвет указывать но только в этот раз указывать со степенью прозрачности

Вторая часть

.modal__block - стилизуем блок который непосредственно будет содержать раскрытую картинку

  • display: flex; - блок делаем флексом
  • flex-direction: column; - выравниваем флексы по вертикали (и почему-то именно этот эффект делает картинку по центру)
  • justify-content: center; - странно то что именно это свойство должно выравнивать флекс по центру но нет
  • align-items: center; - выравнивание элементов по центру (если убрать это свойство картинка будет по центру но растянется по всей ширине экрана)
  • transform: translateY(-200%); - сдвигает элемент по вертикали на указанное значение. Положительное значение сдвигает вниз, отрицательное вверх.
  • transition: all 1s ease; - задаём промежуток времени, в течение которого должен осуществляться переход

.modal__content - стилизуем тег img в модальном окне

  • -webkit-box-shadow: 0px 0px 13px 0px rgba(0,0,0,0.75); - задаем эффект тени элементу
  • -moz-box-shadow: 0px 0px 13px 0px rgba(0,0,0,0.75); - так же задаем эффект тени элементу
  • box-shadow: 0px 0px 13px 0px rgba(0,0,0,0.75); - тоже самое
  • max-width: 100vw; - максимальная ширина 100%
  • max-height: 85vh; - максимальная высота 85%

#caption - блок caption который будет отвечать за подпись под фотографией

  • z-index: 20; - задаем высоту относительно оси перпендикулярной экрану
  • margin: auto; - автоматические отступы делает блок по середине, но для этого надо указать display: block, что мы и делаем в следующем свойстве
  • display: block; - здесь
  • width: 80%; - ширина 80%
  • max-width: 700px; - максимальная ширина 700 пикселей
  • text-align: center; - выравниваем текст по центру
  • color: white; - цыет белый
  • padding: 10px 0; - внутренние отступы сверху и снизу - 10px, а по сторонам - 0
  • height: 150px; - высота 150 пикселей

.modal__content, #caption - устанавливаем анимацию картинке и тексту с описанием картинки

  • animation-name: zoom; - имя нашей анимации
  • animation-duration: .7s; - время выполнения анимации

Третья часть

.close - стилизуем кнопку закрытия картинки

  • z-index: 50; - высота кнопки относительно оси перпендикулярной экрану
  • color: #f1f1f1; - цвет кнопки
  • position: absolute; - абсолютное позиционирование что бы можно было ее выставить поверх окна
  • top:15px; - расстояние до верхней границы контейнера
  • right: 35px; - расстояние до правой границы контейнера
  • font-size: 40px; - размер шрифта
  • font-weight: bold; - жирный шрифт
  • transition: all .3s ease; - задаём промежуток времени, в течение которого должен осуществляться переход
  • cursor: pointer; - выставляем курсор (ладошка с пальцем)

.close:hover, .close:focus - устанавливаем эффекты при наведении и при фокусе на кнопку закрытия

  • color: #bbb; - меняем цвет
  • text-decoration: none; - отключам декорацию
  • cursor: pointer; - устанавливаем курсор (это больше для фокуса видимо)

@keyframes zoom - здесь расписана наша анимация. свойство transform - трансформирует наш элемент, делает он это с помощью функции scale. эта функция масштабирует элемент по горизонтали и вертикали. тоесть нашу анимацию можно прочитать так: от трансформации по масштабу равному 0 (картинка имеет масштам 0 от фактического размера) до трансформации по масштабу равному 1 (картинка имеет масштаб 1 к 1)

Далее идет медиазапрос для адаптации сайта под телефон. Медиазапрос читается так пока максимальный размер ширины экрана не превышает 48em флексы в главном боксе выстраиваются не в строку а в столбец и ширина картинки будет занимать 100% ширины экрана

Здесь весь код помещается в одну функцию которая присваивается событию загрузки окна

В самом начале мы объявляем переменные всех наших элементом. Затем запускаем цикл от i равной 0 до i меньше длине массива imgArr. у нас с классом my__img три элемента. тоесть цикл будет выполнен со значениями i = 0, 1 и 2. Далее каждый тик цикла, переменной picture будет присваиваться значение элемента массива с индексом 0, 1 и 2. тоесть каждыйтик цикла переменная picture станет объектом тега img с каждой картинкой и каждый тик эта переменная будет улетать в функцию openImg.

Что касается слова this, то в этом контексте кода то там можно было написать и так:

openImg(picture)

Теперь рассмотрим функцию openImg. Рассматривать ее лучше на конкретном примере. Допустим у нас прошел первый тик цикла и переменной picture присвоилось значение элемента с первой картинкой. а именно передались такие значения как src и alt. Первое что делает функция это делает наше модальное окно видимым (устанавливает свойство display="block"). далее блоку который непосредственно будет содержать картинку установится свойство transform с нулевым значением (т.е отменили трансформацию). далее тегу img который находится на модальном окне в свойства src и alt присвоили значения из переменной picture. и наконец в caption записали текст с описанием картинки. Интересно сделали, само описание картинки берется из атрибута alt.

Теперь рассмотрим функцию close. тут все предельно просто, элементу с id my__modal (ссылка к нему идет через переменную modalWindow) в свойство display записывается значение none и окно пропадет
